# fletcher-arrow-bridge

Server-side bridge between `eiva-fletcher-core`'s wire format and Apache Arrow C++.
Provides three components in one static library:

- **`Codec`** (`arrow_bridge/codec.hpp`) — binds an `arrow::Schema` to
  `EncodeRow` / `DecodeRow` over the positional wire format. Encodes an
  `ArrowRow` (vector of `arrow::Scalar`) to `EncodedRow` and back. Wire-compatible
  with the positional encoders generated by `fletcher-protoc` for edge code, so
  bytes produced on an embedded device decode cleanly here.
- **`ArrowRowView` / `ArrowScalarList` / `ArrowNestedList`**
  (`arrow_bridge/arrow_row_view.hpp`) — zero-copy template accessors for
  generated row-view classes.
- **CRS utilities** (`arrow_bridge/crs_utils.hpp`) — EPSG → PROJJSON resolution
  and Arrow extension-metadata helpers for geometry fields.

## Building locally

Requires [Conan 2](https://docs.conan.io/2/) and CMake 3.15+.

### Windows

```bash
conan create . -pr:a=Visual-Studio-2022-v143-x64-Debug -o "&:run_tests=True"
```

### Linux (devcontainer)

Open the repository in VS Code and select **Reopen in Container** — the consolidated devcontainer lives at `.devcontainer/` at the repo root and covers every Fletcher component.

```bash
conan create . --build=missing -pr:a=Ubuntu22-gcc-12-Debug -o "&:run_tests=True"
```

## Consuming the package

```python
def requirements(self):
    self.requires("eiva-fletcher-arrow-bridge/0.1.0-alpha")
```

```cmake
find_package(eiva-fletcher-arrow-bridge CONFIG REQUIRED)
target_link_libraries(my-target PRIVATE fletcher::arrow-bridge)
```

```cpp
#include <arrow_bridge/codec.hpp>
#include <arrow_bridge/crs_utils.hpp>
```

`arrow-bridge` re-exports its dependencies (`eiva-fletcher-core`,
`arrow::arrow`) transitively, so consumers don't need to declare them
separately.

## CI pipeline

`.github/workflows/fletcher-arrow-bridge.yml` runs `build-windows` +
`build-linux` on every PR touching `arrow-bridge/**`, and an `upload` job
that publishes platform-specific package binaries to Artifactory on push.
