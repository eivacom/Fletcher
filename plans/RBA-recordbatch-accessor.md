# RBA — RecordBatch Accessor Generator — Execution Plan

Round plan + tracker for generating per-message, column-oriented **RecordBatch
accessors** (C++ and Rust) from `fletcher-protoc`.
Spec (oracle): [docs/recordbatch-accessor-spec.md](../docs/recordbatch-accessor-spec.md).
Locked decisions: [RBA-locked-decisions.md](RBA-locked-decisions.md).
This file is both the `plan_path` (the tracker) and the `user_stories_path`.

## Goal

Generate, per message, a pre-cooked accessor that takes a whole `RecordBatch` in
its constructor, **cheaply verifies it matches the message's schema once**, does
all the column down-casting once up front, and then exposes typed, zero-copy
getters that index straight into the cached concrete column arrays — killing the
`static_pointer_cast<…>(batch->column(i))` boilerplate and the per-cell scalar
allocation of the existing row-oriented `<Class>View`. Emit **C++** and **Rust**,
selected by `--fletcher_opt=accessor` / `--fletcher_opt=rust`. Surface arbitrary
schema/field metadata generically. **Pure add-on** — the existing generator,
consumed in several contexts, does not change behaviour (D-RBA-1).

## Branch strategy

- Branch: **`feature/recordbatchaccessor`**, based on `main` (this feature touches
  only generator-side code + new test harnesses; it does not depend on the
  robustness or DICT work). Per the repo convention, PR branches are **rebased**
  onto `main`, not merged — expect to rebase before the PR.
- No PR until the round is green and reviewed; the PR/merge is the **user's** step.

## Design — generated-code shape

The accessor is **column-oriented**: validate + down-cast every column **once** in
the factory (returning a `Result`, never throwing), then index the cached typed
arrays directly. Validation is **positional + type-only** — it gates on column
count and each column's Arrow *type* (the sole cast-safety invariant), and
deliberately **not** on field name or nullability (oracle §4).

The **same** accessor type is constructible from a `RecordBatch` (top level) **or**
an `arrow::StructArray` — because the two are structurally identical (columns ↔
struct children). **Both factories are generated for every message,
unconditionally** — never gated on whether the message happens to appear as a
nested field — because the same message may be read as a top-level batch in one
context and as a struct column in another, and the generator must not try to
predict which is needed. A nested-message field's getter is **row-bound** — it
returns the nested `<Inner>Accessor`'s `RowView` at the row (built over its struct
column) — so the accessor **composes recursively to any depth**
(`fix.position(row).lat()`, `outer.mid(row).inner().v()`). Cached column arrays own
their buffers, so the accessor keeps **no** reference to the source
`RecordBatch`/`StructArray`.
Representative output for a flat message:

```proto
message Telemetry {
  double          temperature = 1;   // non-nullable scalar
  int32           id          = 2;
  optional string label       = 3;   // nullable scalar
}
```

### Flat example — scalar columns

#### C++ — `<stem>.fletcher.accessor.pb.h`

```cpp
// Generated by fletcher-protoc. DO NOT EDIT.   (#include <arrow/api.h>, etc.)
namespace fletcher_gen { /* namespace <pkg> */ {

/// Column-oriented, read-only accessor over Telemetry's columns. Construct from a
/// RecordBatch (top level) OR an arrow::StructArray (when Telemetry is nested in
/// another message): both go through FromColumns_, which down-casts each column
/// once after a type-only, name-tolerant check. Cached columns own their buffers,
/// so the accessor keeps no reference to the source.
class TelemetryAccessor {
 public:
  static arrow::Result<TelemetryAccessor> Make(const std::shared_ptr<arrow::RecordBatch>& b) {
    return FromColumns_(b->num_rows(), b->columns(),
                        b->schema()->fields(), b->schema()->metadata());
  }
  static arrow::Result<TelemetryAccessor> Make(const std::shared_ptr<arrow::StructArray>& s) {
    arrow::ArrayVector cols;                     // slice each child to the struct's
    for (int i = 0; i < s->num_fields(); ++i)    // [offset,len) window (zero-copy) so
      cols.push_back(s->field(i)->Slice(s->offset(), s->length()));  // row indexing is direct
    return FromColumns_(s->length(), cols, s->struct_type()->fields(), nullptr);
  }

  int64_t num_rows() const { return num_rows_; }

  // Non-nullable scalar getters index the cached typed column directly.
  double  temperature(int64_t row) const { return temperature_->Value(row); }
  int32_t id(int64_t row)          const { return id_->Value(row); }

  // Nullable scalar → std::optional (nullopt when the cell is null).
  std::optional<std::string_view> label(int64_t row) const {
    if (label_->IsNull(row)) return std::nullopt;
    return label_->GetView(row);
  }

  // Generic, domain-agnostic metadata (Fletcher knows none of the keys).
  const arrow::KeyValueMetadata* schema_metadata() const { return schema_metadata_.get(); }
  const arrow::KeyValueMetadata* field_metadata(int i) const { return fields_[i]->metadata().get(); }

 private:
  TelemetryAccessor() = default;

  /// Shared by both factories: type-only gate per column (oracle §4), then cache.
  static arrow::Result<TelemetryAccessor> FromColumns_(
      int64_t num_rows, const arrow::ArrayVector& cols, arrow::FieldVector fields,
      std::shared_ptr<const arrow::KeyValueMetadata> meta) {
    if (cols.size() != 3)
      return arrow::Status::Invalid("Telemetry: expected 3 columns, got ", cols.size());
    // type gate (oracle §4) …
    if (!cols[0]->type()->Equals(*arrow::float64()))
      return arrow::Status::Invalid("Telemetry column 0 'temperature': expected double, got ",
                                    cols[0]->type()->ToString());
    if (!cols[1]->type()->Equals(*arrow::int32())) return arrow::Status::Invalid(/* id */);
    if (!cols[2]->type()->Equals(*arrow::utf8()))  return arrow::Status::Invalid(/* label */);
    // … plus actual-non-nullability for proto-non-nullable fields (temperature, id):
    // a runtime null in a non-nullable field is rejected, so the plain getters below
    // are provably null-free. `label` is proto-`optional` → not checked (its getter
    // returns std::optional). (null_count() is O(1) when there's no validity buffer.)
    if (cols[0]->null_count() != 0)
      return arrow::Status::Invalid("Telemetry column 0 'temperature': non-nullable, found nulls");
    if (cols[1]->null_count() != 0) return arrow::Status::Invalid(/* id non-nullable */);
    TelemetryAccessor a;
    a.num_rows_ = num_rows;
    a.fields_ = std::move(fields);
    a.schema_metadata_ = std::move(meta);
    a.temperature_ = std::static_pointer_cast<arrow::DoubleArray>(cols[0]);  // cached once
    a.id_          = std::static_pointer_cast<arrow::Int32Array>(cols[1]);
    a.label_       = std::static_pointer_cast<arrow::StringArray>(cols[2]);
    return a;
  }

  int64_t num_rows_ = 0;
  arrow::FieldVector fields_;                                     // for *_metadata()
  std::shared_ptr<const arrow::KeyValueMetadata> schema_metadata_;
  std::shared_ptr<arrow::DoubleArray> temperature_;              // cached columns own
  std::shared_ptr<arrow::Int32Array>  id_;                       // their buffers — no
  std::shared_ptr<arrow::StringArray> label_;                    // source ref needed
};

} }  // namespace fletcher_gen::<pkg>
```

#### Rust — `<stem>.fletcher.rs`

```rust
// Generated by fletcher-protoc. DO NOT EDIT.
use std::collections::HashMap;
use std::sync::Arc;
use arrow::array::{Array, ArrayRef, Float64Array, Int32Array, StringArray, StructArray};
use arrow::datatypes::{DataType, Fields};
use arrow::error::ArrowError;
use arrow::record_batch::RecordBatch;

/// Down-cast `col` to `T`, gating on Arrow type only (not name/nullable). Reused
/// for top-level columns and struct children. Uses arrow-rs's offset-preserving,
/// buffer-sharing `downcast_array` (NOT a re-Arc'd downcast_ref clone) so the
/// cached handle keeps the column's offset/len.
fn downcast_array<T: From<arrow::array::ArrayData> + Array + 'static>(
    col: &ArrayRef, ctx: &str, expected: &DataType,
) -> Result<Arc<T>, ArrowError> {
    if col.data_type() != expected {
        return Err(ArrowError::SchemaError(format!(
            "{ctx}: expected {expected:?}, got {:?}", col.data_type())));
    }
    Ok(Arc::new(arrow::array::downcast_array::<T>(col.as_ref())))   // type checked above
}

/// Column-oriented, read-only accessor over Telemetry's columns. Construct from a
/// RecordBatch (top level) OR a StructArray (nested) — both go through
/// from_columns(). Cached columns own their buffers, so no source ref is kept.
pub struct TelemetryAccessor {
    num_rows: usize,
    fields: Fields,                 // for *_metadata()
    metadata: HashMap<String, String>,
    temperature: Arc<Float64Array>,
    id: Arc<Int32Array>,
    label: Arc<StringArray>,
}

impl TelemetryAccessor {
    pub fn try_new(batch: RecordBatch) -> Result<Self, ArrowError> {
        let s = batch.schema();
        Self::from_columns(batch.num_rows(), batch.columns().to_vec(),
                           s.fields().clone(), s.metadata().clone())
    }
    pub fn from_struct(s: &StructArray) -> Result<Self, ArrowError> {
        let DataType::Struct(fields) = s.data_type().clone() else { unreachable!() };
        // Slice each child to the struct's window — columns() are NOT pre-rebased by
        // the struct's logical offset, so a sliced struct would otherwise misalign.
        let cols = s.columns().iter().map(|c| c.slice(s.offset(), s.len())).collect();
        Self::from_columns(s.len(), cols, fields, HashMap::new())
    }

    /// Shared by both ctors: type-only gate per column (oracle §4), then cache.
    fn from_columns(num_rows: usize, cols: Vec<ArrayRef>, fields: Fields,
                    metadata: HashMap<String, String>) -> Result<Self, ArrowError> {
        if cols.len() != 3 {
            return Err(ArrowError::SchemaError(format!(
                "Telemetry: expected 3 columns, got {}", cols.len())));
        }
        // proto-non-nullable fields must hold no actual nulls (oracle §4) → plain
        // getters are null-free; `label` is proto-optional, so it is not checked.
        for (i, name) in [(0usize, "temperature"), (1, "id")] {
            if cols[i].null_count() != 0 {
                return Err(ArrowError::SchemaError(format!(
                    "Telemetry column {i} '{name}': non-nullable, found nulls")));
            }
        }
        let temperature = downcast_array::<Float64Array>(&cols[0], "Telemetry.temperature", &DataType::Float64)?;
        let id          = downcast_array::<Int32Array>(&cols[1], "Telemetry.id", &DataType::Int32)?;
        let label       = downcast_array::<StringArray>(&cols[2], "Telemetry.label", &DataType::Utf8)?;
        Ok(Self { num_rows, fields, metadata, temperature, id, label })
    }

    pub fn num_rows(&self) -> usize { self.num_rows }

    // Non-nullable scalar getters index the cached typed column directly.
    pub fn temperature(&self, row: usize) -> f64  { self.temperature.value(row) }
    pub fn id(&self, row: usize) -> i32           { self.id.value(row) }

    // Nullable scalar → Option.
    pub fn label(&self, row: usize) -> Option<&str> {
        if self.label.is_null(row) { None } else { Some(self.label.value(row)) }
    }

    // Generic, domain-agnostic metadata.
    pub fn schema_metadata(&self) -> &HashMap<String, String> { &self.metadata }
    pub fn field_metadata(&self, i: usize) -> &HashMap<String, String> { self.fields[i].metadata() }
}
```

### Composite example — a nested message that composes the same accessor

A nested message field is a `StructArray` column whose children are the
sub-message's columns. The parent caches the nested message's own `<Inner>Accessor`
(built over that struct column via the `StructArray` factory — validating its
children), and the **struct getter is row-bound**: `fix.position(row)` returns
`PositionAccessor::RowView` (a `{accessor*, row}` forwarder), so you read
`fix.position(row).lat()` — and it composes recursively to any depth. (If
`position` were nullable, the getter returns `std::optional<…RowView>`, None on a
null row.) `Position` below is generated with the identical dual-factory shape as
`TelemetryAccessor` (elided) plus the `RowView`/`at(row)` forwarder shown for
`CoordAccessor` in the collection example, and `Fix` composes it:

```proto
message Position { double lat = 1; double lon = 2; }   // → struct<lat, lon>
message Fix {
  int64    timestamp = 1;
  Position position  = 2;   // a struct column → composes PositionAccessor
}
```

#### C++

```cpp
// PositionAccessor is generated exactly like TelemetryAccessor above — and, like
// EVERY accessor, it gets BOTH factories unconditionally: Make(RecordBatch) AND
// Make(StructArray) → FromColumns_, caching lat_/lon_ as shared_ptr<DoubleArray>,
// with lat(row)/lon(row). (Position can thus be read as a top-level batch on its
// own, or composed as the struct column below.) Body elided.

class FixAccessor {
 public:
  static arrow::Result<FixAccessor> Make(const std::shared_ptr<arrow::RecordBatch>& b) {
    return FromColumns_(b->num_rows(), b->columns(),
                        b->schema()->fields(), b->schema()->metadata());
  }
  static arrow::Result<FixAccessor> Make(const std::shared_ptr<arrow::StructArray>& s) {
    arrow::ArrayVector cols;                     // (so a Fix can itself be nested
    for (int i = 0; i < s->num_fields(); ++i)    //  inside a larger message —
      cols.push_back(s->field(i)->Slice(s->offset(), s->length()));  // any depth)
    return FromColumns_(s->length(), cols, s->struct_type()->fields(), nullptr);
  }

  int64_t num_rows() const { return num_rows_; }
  int64_t timestamp(int64_t row) const { return timestamp_->Value(row); }

  // Struct field is ROW-BOUND: returns the inner accessor's RowView at `row`, so
  // access composes recursively:  fix.position(row).lat()
  PositionAccessor::RowView position(int64_t row) const { return position_->at(row); }
  // If `position` were nullable, this returns std::optional<PositionAccessor::RowView>
  // and is None when the struct column is null at `row` (never reads through null):
  //   std::optional<PositionAccessor::RowView> position(int64_t row) const {
  //     if (position_struct_->IsNull(row)) return std::nullopt;
  //     return position_->at(row);
  //   }

 private:
  FixAccessor() = default;

  static arrow::Result<FixAccessor> FromColumns_(
      int64_t num_rows, const arrow::ArrayVector& cols, arrow::FieldVector fields,
      std::shared_ptr<const arrow::KeyValueMetadata> meta) {
    if (cols.size() != 2)
      return arrow::Status::Invalid("Fix: expected 2 columns, got ", cols.size());
    if (!cols[0]->type()->Equals(*arrow::int64()))               // scalar: type-only via Equals
      return arrow::Status::Invalid("Fix column 0 'timestamp': expected int64, got ",
                                    cols[0]->type()->ToString());
    if (cols[1]->type_id() != arrow::Type::STRUCT)               // struct: shape only here —
      return arrow::Status::Invalid("Fix column 1 'position': expected struct, got ",
                                    cols[1]->type()->ToString());  // children checked by recursion
    FixAccessor a;
    a.num_rows_ = num_rows;
    a.fields_ = std::move(fields);
    a.schema_metadata_ = std::move(meta);
    a.timestamp_ = std::static_pointer_cast<arrow::Int64Array>(cols[0]);
    // Compose the SAME accessor over the struct column; PositionAccessor::Make
    // recursively validates its children (type-only) — to any nesting depth.
    ARROW_ASSIGN_OR_RAISE(
        auto pos, PositionAccessor::Make(std::static_pointer_cast<arrow::StructArray>(cols[1])));
    a.position_ = std::move(pos);
    return a;
  }

  int64_t num_rows_ = 0;
  arrow::FieldVector fields_;
  std::shared_ptr<const arrow::KeyValueMetadata> schema_metadata_;
  std::shared_ptr<arrow::Int64Array> timestamp_;
  std::optional<PositionAccessor> position_;   // built once during Make; same accessor type
};
```

Access composes to any depth — `fix.position(row).lat()`,
`outer.mid(row).inner().v()` — every level being a `<Inner>Accessor` built once
over its struct column and read through a row-bound `RowView`/`Row`. The same
recursion handles a nested struct whose child is itself a struct.

### Collection examples — repeated scalar, repeated struct, map, nested list

Every remaining `FieldKind` is a *collection* column (`ListArray` or `MapArray`).
The recipe is uniform: cache the structural array **and** its flattened typed
child(ren) once; the per-row getter returns a small **span** = `{cached child(ren),
this row's [base, len)}` derived from the offset buffer — **no `GetScalar`, no
allocation**. A struct *element* of a list/map resolves to the inner accessor's
row-bound view (`<Inner>Accessor::RowView` in C++, `<Inner>Accessor::Row` in Rust)
— a `{accessor, flattened-row}` forwarder — so a struct element is still just the
one inner accessor read at the element's flattened index (no second struct type).

The spans are small **library** helpers added in RBA-4 (column-oriented analogues
of the existing per-row `arrow_row_view.hpp` templates). The fixture:

```proto
message Coord  { double x = 1; double y = 2; }
message Ring   { repeated Coord points = 1 [(fletcher.flatten) = true]; }  // flatten-wrapper
message Sample {
  repeated double      readings  = 1;   // REPEATED_SCALAR   → List<double>
  repeated Coord       track     = 2;   // REPEATED_STRUCT   → List<Struct>
  map<string, int32>   counts    = 3;   // MAP scalar→scalar → Map<utf8,int32>
  map<string, Coord>   waypoints = 4;   // MAP scalar→message→ Map<utf8,Struct>
  repeated Ring        rings     = 5;   // NESTED_LIST       → List<List<Struct<Coord>>>
}
```

`Ring` is a **flatten-wrapper** (`(fletcher.flatten)`), so the type mapper absorbs
it: **no `RingAccessor` is generated**, and `repeated Ring` maps to
`List<List<Coord>>` (`FieldKind::NESTED_LIST`, leaf `Coord`). The accessor only
ever sees the absorbed Arrow type, so the leaf reached through `sample.rings(r)` is
a `Coord` (with `.x()`), never a `Ring`. (This is the general rule — flatten-wrappers never get an
accessor or a getter; §1/§2 already skip them from the generated message set.)

#### C++

```cpp
// --- fletcher runtime span helpers (added in RBA-4) -------------------------
// Each span is {cached typed child(ren) + this row's [base,len)} and indexes the
// cached arrays directly. INVARIANT: base/len come from the structural column's
// offset buffer for `row`, and the cached children are the *flattened* values, so
// `child->Value(base+i)` addresses element i of row `row`. STRUCT elements are
// nullable (Arrow list `item` / map `value` are nullable), so they are returned as
// std::optional<RowView> — None when the element is null (no read-through-null,
// closing B2). Scalar elements follow the Arrow norm: `[i]` returns the value and
// `is_null(i)` is available for the caller.
namespace fletcher {
template <class V, class ArrayT> struct ScalarSpan {           // list<scalar>
  const ArrayT* vals; int64_t base, len;
  int64_t size() const { return len; }
  bool    is_null(int64_t i) const { return vals->IsNull(base + i); }
  V       operator[](int64_t i) const { return vals->Value(base + i); }   // + begin/end
};
template <class AccT> struct StructSpan {                       // list<struct>
  const AccT* inner; int64_t base, len;          // inner = accessor over flattened values
  int64_t size() const { return len; }
  bool is_null(int64_t i) const { return inner->is_null(base + i); }
  std::optional<typename AccT::RowView> operator[](int64_t i) const {     // None on a null element
    int64_t r = base + i;
    if (inner->is_null(r)) return std::nullopt;
    return inner->at(r);
  }
};
template <class KV, class KA, class VV, class VA> struct ScalarMapSpan {   // map<scalar,scalar>
  const KA* keys; const VA* vals; int64_t base, len;          // keys non-null; value nullable
  int64_t size() const { return len; }
  KV key(int64_t i)   const { return keys->Value(base + i); }
  bool value_is_null(int64_t i) const { return vals->IsNull(base + i); }
  VV value(int64_t i) const { return vals->Value(base + i); }
};
template <class KV, class KA, class AccT> struct StructMapSpan {           // map<scalar,message>
  const KA* keys; const AccT* vals; int64_t base, len;
  int64_t size() const { return len; }
  KV key(int64_t i) const { return keys->Value(base + i); }
  std::optional<typename AccT::RowView> value(int64_t i) const {          // None on a null value
    int64_t r = base + i;
    if (vals->is_null(r)) return std::nullopt;
    return vals->at(r);
  }
};
template <class AccT> struct NestedStructSpan {                 // list<list<struct>>
  const arrow::ListArray* mid; const AccT* leaf; int64_t base, len;
  int64_t size() const { return len; }
  bool is_null(int64_t i) const { return mid->IsNull(base + i); }       // inner list is nullable too
  std::optional<StructSpan<AccT>> operator[](int64_t i) const {         // None on a null inner list;
    int64_t r = base + i;                                               // its elements are then optional too
    if (mid->IsNull(r)) return std::nullopt;
    return StructSpan<AccT>{ leaf, mid->value_offset(r), mid->value_length(r) };
  }
};
}  // namespace fletcher

// --- the inner reusable struct accessor gains a row-bound forwarder ---------
// RowView BORROWS the accessor — it must not outlive the <Inner>Accessor it points
// at (the accessor owns the cached column buffers). Spans likewise borrow.
class CoordAccessor {                 // dual-factory column accessor (x_/y_ as before)
 public:
  // Make(RecordBatch&) / Make(StructArray&) / num_rows() / x(row) / y(row) … as established.
  // is_null(row): struct-element validity — false when built from a RecordBatch (a
  // batch row is always present); from the StructArray's null bitmap when built via
  // Make(StructArray&), which retains that bitmap (windowed to the struct's slice).
  bool is_null(int64_t row) const { return struct_validity_ && struct_validity_->IsNull(row); }
  struct RowView { const CoordAccessor* a; int64_t row;       // returned for list/map elements
                   double x() const { return a->x(row); }
                   double y() const { return a->y(row); } };
  RowView at(int64_t row) const { return {this, row}; }
 private:
  std::shared_ptr<arrow::StructArray> struct_validity_;   // null when built from a RecordBatch
};

class SampleAccessor {
 public:
  // Make(RecordBatch&) / Make(StructArray&) → FromColumns_; num_rows()/metadata … as established.

  fletcher::ScalarSpan<double, arrow::DoubleArray>                          // REPEATED_SCALAR
  readings(int64_t row) const {
    return { readings_vals_, readings_->value_offset(row), readings_->value_length(row) };
  }
  fletcher::StructSpan<CoordAccessor>                                       // REPEATED_STRUCT
  track(int64_t row) const {            // if (auto e = track(r)[j]) e->x();   (optional element)
    return { &*track_inner_, track_->value_offset(row), track_->value_length(row) };
  }
  fletcher::ScalarMapSpan<std::string_view, arrow::StringArray, int32_t, arrow::Int32Array>
  counts(int64_t row) const {           // MAP scalar→scalar : counts(r).key(j)/.value(j)
    return { counts_keys_, counts_vals_, counts_->value_offset(row), counts_->value_length(row) };
  }
  fletcher::StructMapSpan<std::string_view, arrow::StringArray, CoordAccessor>
  waypoints(int64_t row) const {        // if (auto v = waypoints(r).value(j)) v->x();  (optional value)
    return { waypoints_keys_, &*waypoints_inner_,
             waypoints_->value_offset(row), waypoints_->value_length(row) };
  }
  fletcher::NestedStructSpan<CoordAccessor>                                 // NESTED_LIST
  rings(int64_t row) const {            // rings(r)[i] → optional inner list; (*inner)[j] → optional Coord
    return { rings_mid_, &*rings_leaf_, rings_->value_offset(row), rings_->value_length(row) };
  }

 private:
  // FromColumns_ validates each column type-only + recursively, then caches the
  // structural array + its flattened typed children (and inner accessors):
  static arrow::Result<SampleAccessor> FromColumns_(/* num_rows, cols, fields, meta */) {
    // col 0  readings : list<double>
    //   AsList_(cols[0]) (type_id()==LIST or error); require values()->type()==double()
    // col 1  track : list<struct> → CoordAccessor::Make(list->values() as StructArray)  [recurse]
    // col 2  counts : map ; require keys()->type()==utf8() && items()->type()==int32()
    // col 3  waypoints : map ; require keys()==utf8(); CoordAccessor::Make(items() as Struct) [recurse]
    // col 4  rings : list<list<struct>> → descend two list levels; CoordAccessor::Make(leaf values)
    // then cache (shared_ptr keeps each child's buffers alive):
    //   readings_=l0; readings_vals_=l0->values().get() as DoubleArray*;
    //   track_=l1; track_inner_=…; counts_=m2; counts_keys_/counts_vals_=…;
    //   waypoints_=m3; waypoints_keys_/waypoints_inner_=…;
    //   rings_=l4; rings_mid_=l4->values().get() as ListArray*; rings_leaf_=…
  }

  std::shared_ptr<arrow::ListArray> readings_;  const arrow::DoubleArray* readings_vals_ = nullptr;
  std::shared_ptr<arrow::ListArray> track_;     std::optional<CoordAccessor> track_inner_;
  std::shared_ptr<arrow::MapArray>  counts_;    const arrow::StringArray* counts_keys_ = nullptr;
                                                const arrow::Int32Array*  counts_vals_ = nullptr;
  std::shared_ptr<arrow::MapArray>  waypoints_; const arrow::StringArray* waypoints_keys_ = nullptr;
                                                std::optional<CoordAccessor> waypoints_inner_;
  std::shared_ptr<arrow::ListArray> rings_;     const arrow::ListArray* rings_mid_ = nullptr;
                                                std::optional<CoordAccessor> rings_leaf_;
};

// --- fletcher runtime span helpers (added in RBA-6) -------------------------
// arrow-rs's ArrayAccessor is implemented on the *reference* (&Float64Array,
// &StringArray, …), not the owned array — so the span's type param A is the
// reference type, bound `A: ArrayAccessor` and held by value (it already carries
// the lifetime). One generic ScalarSpan then covers every scalar array.
use arrow::array::{Array, ArrayAccessor};

pub struct ScalarSpan<A: ArrayAccessor> { vals: A, base: usize, len: usize }      // A = &'a Float64Array, …
impl<A: ArrayAccessor> ScalarSpan<A> {
    pub fn len(&self) -> usize { self.len }
    pub fn is_null(&self, i: usize) -> bool { self.vals.is_null(self.base + i) }
    pub fn value(&self, i: usize) -> A::Item { self.vals.value(self.base + i) }   // f64, &str, …
}
// STRUCT elements are nullable, so get(i)/value(i) return Option<Acc::Row> — None
// on a null element (no read-through-null, closing B2). `Acc` exposes is_null(row)
// and row(i)->Row (e.g. CoordAccessor below).
pub struct StructSpan<'a, Acc: RowAccess> { inner: &'a Acc, base: usize, len: usize }   // list<struct>
impl<'a, Acc: RowAccess> StructSpan<'a, Acc> {
    pub fn len(&self) -> usize { self.len }
    pub fn is_null(&self, i: usize) -> bool { self.inner.is_null(self.base + i) }
    pub fn get(&self, i: usize) -> Option<Acc::Row<'_>> {
        let r = self.base + i;
        if self.inner.is_null(r) { None } else { Some(self.inner.row(r)) }
    }
}
pub struct ScalarMapSpan<K: ArrayAccessor, V: ArrayAccessor> {                    // map<scalar,scalar>
    keys: K, vals: V, base: usize, len: usize,                                    // K=&'a StringArray, V=&'a Int32Array
}                                                                                 // keys non-null; value nullable
pub struct StructMapSpan<'a, K: ArrayAccessor, Acc: RowAccess> {                  // map<scalar,message>
    keys: K, vals: &'a Acc, base: usize, len: usize,
}
impl<'a, K: ArrayAccessor, Acc: RowAccess> StructMapSpan<'a, K, Acc> {
    pub fn key(&self, i: usize) -> K::Item { self.keys.value(self.base + i) }
    pub fn value(&self, i: usize) -> Option<Acc::Row<'_>> {                       // None on a null value
        let r = self.base + i;
        if self.vals.is_null(r) { None } else { Some(self.vals.row(r)) }
    }
}
pub struct NestedStructSpan<'a, Acc: RowAccess> { mid: &'a ListArray, leaf: &'a Acc, base: usize, len: usize }
impl<'a, Acc: RowAccess> NestedStructSpan<'a, Acc> {
    pub fn len(&self) -> usize { self.len }
    pub fn get(&self, i: usize) -> Option<StructSpan<'a, Acc>> {        // None on a null inner list
        let r = self.base + i;
        if self.mid.is_null(r) { return None; }                        // inner list is nullable too
        let o = self.mid.value_offsets();
        Some(StructSpan { inner: self.leaf, base: o[r] as usize, len: (o[r + 1] - o[r]) as usize })
    }
}
// All spans BORROW the cached children/accessor — they (and Rows) must not outlive the SampleAccessor.
// Construction sites pass the reference, e.g. ScalarSpan { vals: &*self.readings_vals, … }.

// --- the inner reusable struct accessor: row-bound forwarder + null/validity ----
// RowAccess lets the generic spans get a Row and test element validity uniformly.
pub trait RowAccess { type Row<'a> where Self: 'a; fn row(&self, i: usize) -> Self::Row<'_>;
                      fn is_null(&self, row: usize) -> bool; }
pub struct CoordRow<'a> { a: &'a CoordAccessor, row: usize }
impl CoordRow<'_> { pub fn x(&self) -> f64 { self.a.x(self.row) }
                    pub fn y(&self) -> f64 { self.a.y(self.row) } }
impl RowAccess for CoordAccessor {    // dual-ctor column accessor (x/y as before)
    type Row<'a> = CoordRow<'a>;
    fn row(&self, i: usize) -> CoordRow<'_> { CoordRow { a: self, row: i } }
    // false when built from a RecordBatch; from the StructArray's null bitmap (kept,
    // windowed to the struct slice) when built via from_struct.
    fn is_null(&self, row: usize) -> bool {
        self.struct_validity.as_ref().map_or(false, |s| s.is_null(row))
    }
}

pub struct SampleAccessor {
    num_rows: usize, fields: Fields, metadata: HashMap<String, String>,
    readings: Arc<ListArray>,  readings_vals: Arc<Float64Array>,        // REPEATED_SCALAR
    track: Arc<ListArray>,     track_inner: CoordAccessor,              // REPEATED_STRUCT
    counts: Arc<MapArray>,     counts_keys: Arc<StringArray>, counts_vals: Arc<Int32Array>,   // MAP s→s
    waypoints: Arc<MapArray>,  wp_keys: Arc<StringArray>,    wp_inner: CoordAccessor,         // MAP s→msg
    rings: Arc<ListArray>,     rings_mid: Arc<ListArray>,    rings_leaf: CoordAccessor,        // NESTED_LIST
}

impl SampleAccessor {
    // try_new(RecordBatch) / from_struct(&StructArray) → from_columns (as established).
    // from_columns validates each column type-only + recursively, then caches:
    //   readings: list → values() must be Float64;  track: list → CoordAccessor::from_struct(values)
    //   counts: map → keys Utf8, values Int32;       waypoints: map → keys Utf8, CoordAccessor on values
    //   rings: list<list<struct>> → descend two levels → CoordAccessor on the leaf values

    // list() and map both use 32-bit, non-negative, monotonic offsets (large_list /
    // large_utf8 out of scope, §10), so i32 → usize cannot wrap. MapArray is NOT a
    // ListArray in arrow-rs, so it gets its own bounds helper over MapArray::value_offsets().
    fn list_bounds(l: &ListArray, row: usize) -> (usize, usize) {
        let o = l.value_offsets();
        (o[row] as usize, (o[row + 1] - o[row]) as usize)
    }
    fn map_bounds(m: &MapArray, row: usize) -> (usize, usize) {           // entry [base, len)
        let o = m.value_offsets();
        (o[row] as usize, (o[row + 1] - o[row]) as usize)
    }

    pub fn readings(&self, row: usize) -> ScalarSpan<&'_ Float64Array> {
        let (base, len) = Self::list_bounds(&self.readings, row);
        ScalarSpan { vals: &*self.readings_vals, base, len }            // &Float64Array: ArrayAccessor
    }
    pub fn track(&self, row: usize) -> StructSpan<'_, CoordAccessor> {    // if let Some(e)=track(r).get(j) { e.x() }
        let (base, len) = Self::list_bounds(&self.track, row);
        StructSpan { inner: &self.track_inner, base, len }
    }
    pub fn counts(&self, row: usize) -> ScalarMapSpan<&'_ StringArray, &'_ Int32Array> {
        let (base, len) = Self::map_bounds(&self.counts, row);           // MapArray bounds, not list
        ScalarMapSpan { keys: &*self.counts_keys, vals: &*self.counts_vals, base, len }
    }
    pub fn waypoints(&self, row: usize) -> StructMapSpan<'_, &'_ StringArray, CoordAccessor> {
        let (base, len) = Self::map_bounds(&self.waypoints, row);        // if let Some(v)=…value(j) { v.x() }
        StructMapSpan { keys: &*self.wp_keys, vals: &self.wp_inner, base, len }
    }
    pub fn rings(&self, row: usize) -> NestedStructSpan<'_, CoordAccessor> {   // rings(r).get(i)?.get(j)?.x()
        let (base, len) = Self::list_bounds(&self.rings, row);
        NestedStructSpan { mid: &self.rings_mid, leaf: &self.rings_leaf, base, len }
    }
}
```

Usage: scalar elements read directly (`sample.readings(r)[k]`,
`sample.counts(r).value(j)`); struct elements come back as **optionals** (None on a
null element), so:
- C++ — `if (auto e = sample.track(r)[j]) e->x();`,
  `if (auto v = sample.waypoints(r).value(j)) v->x();`,
  `if (auto inner = sample.rings(r)[i]) if (auto e = (*inner)[j]) e->x();`
  (the inner list level is itself optional — None on a null inner list)
- Rust — `if let Some(e) = sample.track(r).get(j) { e.x() }`,
  `if let Some(v) = sample.waypoints(r).value(j) { v.x() }`,
  `sample.rings(r).get(i).and_then(|inner| inner.get(j)).map(|e| e.x())`

Every struct element (`track`, `waypoints` value, `rings` leaf) reuses the one
`CoordAccessor` read at a flattened index — never a second struct type, never a
`GetScalar`, and never read through a null.

> The samples above are illustrative of the **shape**, not final text. Exact
> arrow-rs API spellings (the downcast helpers, `value`/`is_null`, offset/metadata
> accessors, the `ArrayAccessor` bound, the span lifetimes) and the pinned crate
> version are fixed at implementation (RBA-5/RBA-6). The span helper types
> (`ScalarSpan`/`StructSpan`/`*MapSpan`/`NestedStructSpan`) are small library
> additions — the column-oriented analogues of the existing per-row
> `arrow_row_view.hpp` templates — instantiated by the generated getters.

## Items (user stories + acceptance)

### RBA-1 — Option plumbing + additive no-drift guarantee

**Story.** As a Fletcher maintainer I can pass `--fletcher_opt=accessor` and/or
`rust` and get new accessor output **files**, while every existing output stays
byte-identical, so the feature is provably a pure add-on (D-RBA-1, D-RBA-2).

**Scope.**
- `generator.cpp` `ArrowRowGenerator::Generate`: parse `accessor` and `rust`
  tokens alongside `ts`/`ipc`/`schema_only`; when set, `context->Open` the new
  output filenames (`<stem>.fletcher.accessor.pb.h`, `<stem>.fletcher.rs`) and
  write them via **new** emitter entry points. Initial cut may emit a minimal but
  valid file (header guard / module banner) — content lands in RBA-2+.
- Add the new emitter source/header(s) without editing existing emitter
  functions; reuse `OrderedMessages`/`GatherFields`/`FieldInfo` read-only.

**Forcing test** (`AccessorTest.OptGatedEmissionLeavesExistingOutputsByteIdentical`,
integration harness): the no-drift gate must cover the **full existing option
matrix**, not one run. For each baseline opt set in
`{∅, ts, ipc, schema_only, ts,ipc, schema_only,ts,ipc}`, run the plugin twice —
once with that set, once with that set **plus** `accessor,rust` — and assert every
file the baseline produced (`.fletcher.pb.h`, `.fletcher.arrow.pb.h`, and the
`.fletcher.ts` / `.<Message>.ipc` files when those opts are on) is **byte-identical**
between the two runs, and that the `accessor,rust` run produced *only* the two new
files in addition. Proven red-first by introducing the assertion before the opt
plumbing. (This is what makes D-RBA-1 falsifiable: a single-baseline check could
miss drift that only appears under, say, `schema_only` or `ipc`.)

**Acceptance.** Unknown/absent opt ⇒ no new files. New files are well-formed
(C++ header compiles empty; Rust module parses). No existing test output changes.

### RBA-2 — C++ scalar accessor + positional type-check validation

**Story.** As a C++ consumer I construct `<Class>Accessor::Make(batch)` for a flat
message and read typed scalar columns directly, with a one-time, name-tolerant
schema check (D-RBA-3, D-RBA-4, D-RBA-7).

**Scope.**
- Emit the `<Class>Accessor` class per the §3/§4/§7 shape, with **both** factories
  sharing one `FromColumns_` helper: `Make(const shared_ptr<RecordBatch>&)` and
  `Make(const shared_ptr<StructArray>&)`, each `-> arrow::Result<…>`. `FromColumns_`
  checks column count + per-column `type()->Equals(expected, /*ignore metadata*/)`,
  caches the columns as `std::shared_ptr<T>` (buffers stay alive — no source ref),
  and stores the `FieldVector` + schema metadata. Non-nullable scalar getters
  return the value type; nullable scalar getters return `std::optional<…>`
  (string/binary → `std::string_view`); plus `num_rows()`. **Both factories are
  generated for every message unconditionally** (D-RBA-7) — the `StructArray`
  factory is never gated on whether the message is nested; it is the basis for
  nested composition in RBA-4 and is also directly callable. RBA-2's test exercises
  both factories (build a `RecordBatch` and build an equivalent `StructArray`).
- Cover all scalar Arrow types the type-mapper produces (bool/int/uint/float/
  double/utf8/binary/timestamp/duration), reusing `ScalarTypeInfo`.

**Forcing test** (`AccessorTest.ScalarColumnsReadAndValidatePositionally`): build a
`RecordBatch` for a scalar fixture; assert each getter returns the right per-row
value (incl. a *nullable* field with a null → `nullopt`); a **type-mismatched**
batch → failed `Result` with a precise message; a **name-mismatched but
type-compatible** batch → **success** (the name-tolerance proof, oracle §4); and a
**proto-non-nullable scalar column that carries a runtime null** → failed `Result`
(the `null_count==0` gate, D-RBA-4 — proven here at the cheapest layer).

**Acceptance.** Wrong column count → error; `Make` never throws; the accessor
keeps its data alive via the cached column handles (read after the caller drops
every other ref to the batch); the `Make(StructArray)` factory reads an equivalent
`StructArray` identically (both factories present and working on a flat message);
a non-nullable column with **no** validity buffer is accepted (the null check is
`null_count==0`, not "has a validity buffer").

### RBA-3 — C++ generic metadata access

**Story.** As a consumer I read arbitrary schema/field metadata off the accessor,

**Scope.** Add `schema_metadata()` and `field_metadata(i)` / `field_metadata(name)`

**Forcing test** (`AccessorTest.ExposesSchemaAndFieldMetadataGenerically`): build a

### RBA-4 — C++ full type parity (struct / list / map / nested)

**Story.** As a consumer I read nested and repeated columns column-oriented, at
parity with `<Class>View` (D-RBA-6).

**Scope.** Extend the C++ accessor to `STRUCT`, `REPEATED_SCALAR`,
`REPEATED_STRUCT`, `MAP`, and `NESTED_LIST` (depth 2–3), plus the `RowView`
forwarder + `at(row)` on every accessor (D-RBA-7).
- A **nested-message (`STRUCT`) field** caches the nested `<Inner>Accessor` (built
  via the `StructArray` factory from RBA-2 — which **slices each child to the
  struct's `[offset,len)` window** (B1) and recursively validates child types). The
  getter is **row-bound**: `position(row)` → `<Inner>Accessor::RowView`, or
  `std::optional<…RowView>` when nullable — **None on a null row** (never reads
  through null). Composes recursively: `a.outer(row).inner().leaf()`.
- `REPEATED_SCALAR`/`REPEATED_STRUCT`/`MAP`/`NESTED_LIST` cache the corresponding
  `ListArray`/`MapArray` (+ flattened typed children / inner accessor) and return
  spans (`std::optional<Span>` when the field is row-nullable); struct elements are
  `<Inner>Accessor::RowView`.
- **Cross-file (B5/D-RBA-10):** when a nested message comes from an imported
  `.proto`, emit the cross-file `#include` via the existing `CollectCrossFileIncludes`,
  mapping `.fletcher.pb.h` → `.fletcher.accessor.pb.h`.
- May split into RBA-4a (struct + lists, incl. RowView + cross-file) / RBA-4b
  (maps + nested lists) if heavy.

**Forcing test** (`AccessorTest.CompositeColumnsReadColumnOriented`): a fixture
exercising **every composite kind**, read column-oriented over a real `RecordBatch`:
- a nested struct **whose child is itself a struct, imported from a second `.proto`**
  (`STRUCT`, ≥2 levels + cross-file), asserting the deep chain
  `a.outer(row).inner().leaf()` through composed `RowView`s;
- a `REPEATED_SCALAR`, a `REPEATED_STRUCT`, a `NESTED_LIST`;
- **both** map value-types — `map<…, scalar>` (`ScalarMapSpan`) **and**
  `map<…, message>` (`StructMapSpan`).

**Null-safety is part of the gate** (the cycle 2–3 hardening — assert each):
- a **nullable 1:1 struct field**, null row → `std::nullopt`;
- a **null struct element** in the repeated-struct → `span[j] == std::nullopt`;
- a **null map message value** → `value(j) == std::nullopt`;
- a **null inner list** in the nested list → `span[i] == std::nullopt`;
- a **non-nullable struct field with a runtime null** → `Make` returns an error
  `Result` (`null_count` gate recursed into composites, D-RBA-4).

**Acceptance tests** cover empty list/map, scalar element nulls via `is_null(j)`,
name-tolerance on a nested/cross-file field, and an unsupported construct emitting
the documented comment.

  files sharing a module. The Cargo crate gains a **two-file, two-package import
  fixture** that exercises it.

**Forcing test** (`composite_and_metadata_read`, `cargo test`): the Rust mirror of
RBA-4's strengthened test — every composite kind plus the null-safety paths:
- a nested struct **whose child is itself a struct imported from a second `.proto`**

- null-safety: nullable 1:1 struct row → `None`; null struct element

- plus arbitrary schema/field metadata read-back.

**Acceptance.** Matches RBA-3/RBA-4 acceptance, in Rust (incl. no read-through-null,
